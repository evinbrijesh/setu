# Setu — User Stories / Feature List

## Epic A — Voice Intake
- **US-A1:** As a user, I want to speak my request in Hindi, Tamil, or Hinglish, so that I don't need to type or read a form. *(MVP)*
- **US-A2:** As a user, I want the agent to understand what government service I'm asking about from natural phrasing, so that I don't need to know the official name of the process. *(MVP)*
- **US-A3:** As a user, I want to hear the agent's questions spoken back to me naturally, so that the entire interaction stays voice-only. *(MVP)*

## Epic B — Guided Information Collection
- **US-B1:** As a user, I want the agent to ask me for one piece of information at a time, so that I'm not overwhelmed by a long list of requirements. *(MVP)*
- **US-B2:** As a user, I want the agent to tell me clearly what's still missing, so that I know exactly what to provide next. *(MVP)*
- **US-B3:** As a user, if the agent misunderstands my answer, I want to be asked to clarify rather than have incorrect information silently recorded. *(MVP)*

## Epic C — Durable, Resumable Process
- **US-C1:** As a user, I want to stop the conversation at any point and come back later, so that I'm not forced to complete everything in one sitting. *(MVP — core differentiator)*
- **US-C2:** As a user, when I return, I want the agent to recall everything I've already told it, so that I never repeat myself. *(MVP)*
- **US-C3:** As a builder/judge, I want to see the underlying process represented as discrete, resumable stages with visible execution history. *(MVP)*

## Epic D — Validation and Output
- **US-D1:** As a user, I want the agent to tell me whether I'm eligible for a scheme based on what I've provided. *(MVP)*
- **US-D2:** As a user, once all information is collected, I want a completed, downloadable form. *(MVP)*
- **US-D3:** As a user, I want to be notified (voice and/or text) when my document is ready. *(MVP)*

## Epic E — Visibility and Trust
- **US-E1:** As a user, I want to see a live progress indicator ("Step 3 of 5"). *(MVP)*
- **US-E2:** As a judge/demo viewer, I want to see the actual execution graph of a run (task-by-task). *(MVP)*
- **US-E3:** As a user, I want to switch between at least two languages mid-use. *(MVP)*

## Epic F — Stretch Goals (only if ahead of schedule)
- **US-F1:** Apply for a second, different government scheme through the same interface.
- **US-F2:** Interact via an actual phone call rather than only a browser microphone.
- **US-F3:** Basic authenticity checks on documents mentioned.

## Prioritization Summary

| Priority | Stories |
|---|---|
| Must-have (MVP) | A1–A3, B1–B3, C1–C3, D1–D3, E1–E3 |
| Stretch | F1–F3 |

Phases 0–6 deliver every MVP story. Phase 7 (second scheme) maps to F1. F2/F3 out of scope for this build.
